# Sample Hardhat Project

This project demonstrates a basic Hardhat use case. It comes with a sample contract, a test for that contract, and a Hardhat Ignition module that deploys that contract.

Try running some of the following tasks:

```shell
npx hardhat help
npx hardhat test
REPORT_GAS=true npx hardhat test
npx hardhat node
npx hardhat ignition deploy ./ignition/modules/Lock.js
```

```
Убедитесь, что на компьютере установлены Node.js и Python, а в проекте установлены все зависимости:

    Для блокчейна: npm install

    Для Python: pip install web3

Шаг 1. Запуск локального блокчейна (Сердце системы)

Любая демонстрация начинается с запуска собственной сети.

    Откройте VS Code и запустите Терминал 1.

    Введите команду:
    Bash

    npx hardhat node

    Важно: Оставьте этот терминал открытым на протяжении всей демонстрации. Блокчейн должен работать в фоновом режиме.

Шаг 2. Сценарий №1: Демонстрация защищенной площадки

Этот сценарий показывает, как работает надежный смарт-контракт с защитой от атак (Мьютекс) и криптографической подписью транзакций.

    Откройте Терминал 2 (нажмите + в панели терминалов).

    Выполните скрипт развертывания защищенной площадки:
    Bash

    npx hardhat run scripts/deploy.js --network localhost

    Скопируйте адрес контракта, который выдаст терминал.

    Откройте файл app.js.

    Вставьте скопированный адрес в самую первую строку: var CONTRACT_ADDRESS = "ВАШ_АДРЕС"; и сохраните файл (Ctrl + S).

    Откройте файл index.html в любом браузере.

    Продемонстрируйте комиссии пополнение баланса (кнопка "Внести") и снятие средств (кнопка "Вывести"). Обратите внимание комиссии на логи: транзакции подписываются локально и успешно проходят проверки безопасности.

Шаг 3. Сценарий №2: Демонстрация хакерской атаки (Reentrancy)

Этот сценарий показывает, что происходит, если смарт-контракт написан с ошибками логики (отправка средств до обнуления баланса), и как хакер может выкачать все деньги.

    В Терминале 2 запустите скрипт подготовки уязвимого стенда:
    Bash

    npx hardhat run scripts/setup_hack.js --network localhost

    Скрипт развернет площадку, начислит туда 6 ETH от пользователей и развернет контракт хакера. Терминал выдаст два адреса.

    Скопируйте эти два адреса.

    Откройте файл hacker.html.

    Вставьте адреса в переменные TARGET_MARKET и ATTACKER_CONTRACT внутри тега <script> и сохраните файл (Ctrl + S).

    Откройте файл hacker.html в браузере.

    Нажмите кнопку [ EXECUTE EXPLOIT ].

    Обратите внимание комиссии на терминал логов: хакер перехватывает управление через fallback функцию и зацикливает вывод средств, пока баланс площадки не станет равен нулю.

Шаг 4. Сценарий №3: Мониторинг через Python (Опционально)

Демонстрация интеграции блокчейна с внешними скриптами на Python.

    Убедитесь, что в скрипте market_reader.py указан правильный адрес кошелька, баланс которого вы хотите проверить.

    В Терминале 2 выполните команду:
    Bash

    python market_reader.py

    Скрипт подключится к локальному узлу (127.0.0.1:8545) и выведет актуальные данные из блокчейна напрямую в консоль.

 Частые ошибки на защите (Troubleshooting)

    Транзакция отклоняется сразу (Reverted): Вы забыли запустить npx hardhat node в первом терминале, либо нода "зависла". Просто закройте первый терминал и запустите ноду заново.

    Ничего не происходит после нажатия кнопки в браузере: Вы забыли нажать Ctrl + S после вставки новых адресов в файлы app.js или hacker.html. Сохраните файл и обновите страницу в браузере (F5).

    Ошибка компиляции "Identifier not found": Убедитесь, что перед запуском вы выполнили команду npx hardhat compile после всех изменений в смарт-контрактах.
    ```
